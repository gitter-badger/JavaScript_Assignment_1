Chapter 01 - Basics of JavaScript

What are some cross-browser issues that affect JavaScript code?
Answer: Web browsers rely on two distinct engines to load and render a webpage.
  • JavaScript Engine
  • Rendering Engine
The inconsistent implementation of web standards by various browser vendors of these engines make it nearly impossible to be absolutely sure if the code we author will behave and appear the same way on every platform. The DOM API provided by each of these vendors differs ever so slightly making it difficult to write simple JavaScript code without having to include shims and polyfills or some sort of fallback mechanism
What tools are available to deal with these issues?
Answer: It serves us well in general to follow these guidelines to get around these cross-browser issues
  • Don't leave anything to the browser to make assumptions. Instead be explicit wherever possible. Ex: include the DOCTYPE for every webpage
  • Ensure the HTML document is valid and is wellformed
  • Rely on popular JavaScript libraries and CSS frameworks that normalize the cross-browser idiosyncrasies.
  • Use popular unit-testing tools such as Jasmine, mocha-chai to run tests in all the browsers using a testrunner such as Karma
What considerations should we take when determining what browsers to support?
Answer: Although various strategies exist, the one that Yahoo follows seems to be more popular. It involves putting together a matrix which Yahoo calls the "graded browser matrix" where the browsers are along one axis and the platforms are along the second axis. We then need to grade them depending on the project requirements
What sort of testing should we do on JavaScript code before deployment?
Answer: Before deploying the JavaScript code, we should be certain that the test cases we author all pass and posses the three main characteristics, namely
  1. Repeatability
  2. Simplicity
  3. Isolation/Independent
Types of testing would include
  • functionality
  • performance
  • compatibility
, There are two approaches to testing the code we write when we bump into a bug and we want to debug it.
  1. Constructive
  2. Deconstructive

02 - Testing Frameworks

What debug tools can we use for JavaScript?
Answer: We can use google chrome developer tools to place break points and perform stepwise execution to debug JavaScript. Also we could use the console.log() method to log to the console.
Explain how deconstructive and constructive test cases work.
Answer: Deconstructive test cases are composed by isolating just the code that constitute the system-under-test and eleminating the rest of the code that do not participate in the test cases. Constructive test cases on the other hand are put together by starting from a known good reduced case and go on adding code to it until a bug shows up.
What is the purpose of a testing framework?
Answer: A good testing framework enables us to organize our tests by grouping them together, allow us to see a dashboard of tests that have passed and the ones that need fixing. When it comes to JavaScript Unit-Testing frameworks aimed at the DOM API, we need a test-framework that can
  1. simulate browser behavior such as mouse-hover, clicks, drag and drop
  2. allow us to control the test runner interactively
  3. give us the ability to filter what tests are to be executed
  4. provide a way to test asynchronous operations by grouping all those that depend on the same asynchronous operation

Chapter 03 - Functions

Explain how functions work in JavaScript
Answer:
  • JavaScript functions can be invoked in one of four ways.
    1. as a function
    2. as a method on an object
    3. as an object constructor
    4. as apply/call
  • Every function is implicitly provided with two additional parameters, namely "this" and "arguments". The "this" parameter is influenced by above the invokation patterns (listed above) used
  • Additionally, JavaScript functions are basically objects (mutable key-value pairs) and are instances of the core JavaScript object named "Function" Therefore every function has a hidden link to "Function.prototype". Which means that anything attached to the Function.prototype instantly becomes available to all instances the functions.
  • Every function has a property called "prototype" which is an object consisting of a property called "constructor" whose value is the function itself.
  • functions can be stored in variables, objects, arrays, they can be passed to other functions as arguments and returned from other functions as values.
    • How are they declared?
    Answer: We could use one of two syntaxes to declare JavaScript functions.
    • function declaration/statement syntax where the function declaration is hoisted to the top of the function scope.
      code
      example: function bar(){ //code here }
    • function expression sytax(the preferred way) where the function declaration is not hoisted and can only be called after the function has been defined.
      code
      example: var bar = function bar(){ //code here } OR var bar = function (){ //code here }
    Where can they be used?
    Answer:
    • functions can be used to create objects when invoked with a new keyword
    • functions can be used to implement callbacks that could be either synchronous or asynchronous
    • A function can be executed within the context of an object as a method
    • functions that are in the outermost scope are actually methods on the global object. In the case of browser, the functions are methods on the global "window" object. However, they can be invoked without referencing the global object.
    • functions can be stored in variables, objects, arrays.
    • They can be passed to other functions as arguments and can be returned from functions.
    How do we call them?
    Answer: There are several ways of calling a function
    • As a regular function. example: add(3,4);
    • As a method on an object.example : myObj.add(3,4)
    • As a constructor. example: var cal = new Calculater();
    • With a Apply/Call invocation (which allows setting the context for the method).example add.call(MyObj,3,5); or add.apply(MyObj,[3,5]);
    • a function may also be invocked as follos var bar = function(a,b){ return a + b;}; bar.prototype.constructor();
    Are there any special characteristics that differentiate them from other programming languages’ functions?
    Answer:
    • JavaScript functions are objects. Which means that they can in-turn have methods and properties. Which makes it possible to memoize previously computed expensive operations, thereby making it possible to author performant code
    • JavaScript functions have the ability to dynamically bind the execution context with object using "apply" and "call" pattern of invocation.
    • By default, all JavaScript functions receive two additional bonus parameters, namely "this" and "arugments". The arguments parameter makes it possible to turn the function into one that can accept var-args
    How are parameters handled by functions?
    Answer: If a function is called by passing in more arguments than it can take, the extra arguments are ignored and if a function is called with fewer arguments than it takes, the called function proceeds with the execution by sticking in "undefined" for those parameters that didn't receive any value.
    Can we define var-args functions?
    Answer: We can achieve this by leveraging the implicitly passed bonus parameter called "arguments". It is basically an array-like object which holds the parameters passed to it as arguments from the calling function.
    How can we define a variable’s type?
    Answer: As JavaScript is a dynamically-typed language, the variable's type is determined by the type of value we assign to it
    How are variables declared?
    Answer: variables are declared with a "var" keyword;
    What is the purpose of the var keyword?
    Answer: "var" keyword is used to declare a variable. In ES5, the use of var keyword for a variable inside a function, influences it's scope. When "use strict" is not used, forgetting to use the var keyword for a variable intended as a local variable will cause to silently be scoped to its outer namespace.
    Describe briefly how JavaScript code is executed on the browser.
    Answer: Within the browser, the JavaScript code is interpreted and executed by the JavaScript Runtime Engine. Although we can author and execute JavaScript code without involving the DOM API, it would be meaningless to do so as we most certainly would want our code to work with the DOM. When coding against the DOM API, the code we author is hooked in as eventhandlers. These eventhandlers are nothing but callback functions that are queued in on the "eventloop" (that runs on a single-thread) and executed in a first-come-first-served basis like any good queues would. The thread that registers events on the "eventloop" is separate from the one on which the eventloop itself runs.
    What is a callback?
    Answer: To my mind, a JavaScript callback is what C programming langauge calls "function pointer" and C# calls a "delegate". It is basically a function that gets executed at a later time when it's turn matures on the eventloop queue (asynchronous callback) or is simply executed by some other code (synchronous callback). We can't tell if a callback in question is synchronous or asynchronous until we consult the API documentation.
    example of syncrhonous callback; var bar = function(){ return baz(function(msg){ console.log(msg + ' Shardul'); }); }; var baz = function(cb){ return cb('Ola'); }; /*Another simple example of a synchronous callback would be Array.sort()*/ var myArray = [23,44,10,9,5,20]; myArray.sort(function(a,b){ return a - b; });
    How does scoping work in JavaScript?
    Answer: In JavaScript, the code is always scoped to "function" and not "block". The variables that are declared (with a var keyword) within a function context stay local to it and are inaccessible outside. Skipping to declare it within the function (with a var keyword) will cause it to be implicitly considered as a global variable.It is thus a good practice to always include 'use strict'; as the first line in every top-level function to enforce declaring local variables with a var keyword. Failing to declare it with a var keyword will cause it to error out (with an uncaught ReferenceError), unless the variable is declared in the outer scope.
    What is Hoisting?
    Answer: The variable declaration in JavaScript is subject to "hoisting" which simply means, all declarations are moved to the top of the function by the interpreter. When it comes to functions, the function definitions (also known as statements) are hoisted to the top of the function and not function expressions
    How does the context of a function work in JavaScript?
    Answer: The context of a JavaScript function is nothing but the object on which the function will be attached to as a method and when invoked, will behave like a any other method on that object.
    Is it possible to alter it? If so, how?
    Answer: Yes it is. We can do so using either apply or call methods that exist on every function, like so
    var setName = function(fn,ln){ this.firstName = fn; this.lastName = ln; }; var Person = {}; setName.call(Person,'Shardul','Ghanti'); // or setName.apply(Person,['Shardul','Ghanti']); console.log(Person.firstName + ' ' + Person.lastName);
    What happens when a function is called using the new keyword?
    Answer: When a JavaScript function is called with a new keyword,the function is executed as an object constructor and a new object is created on the heap. If there are properties and methods attached to it's prototype,they become available to this new object created.
    Explain how the this keyword works in JavaScript.
    Answer: the this keyword refers to the execution context of the function. The execution context of a function is influenced by how a function is invoked.
    S.No Invokation Type 'this keyword'
    1 function 'this' refers to the global object.
    2 method 'this' refers to the object, the method is called on.
    3 constructor 'this' refers to the new object being created.
    4 apply/call 'this' refers to the object passed in as the first parameter.
    How can the return type of a function be specified?
    Answer: The return type of a function is determined by what we choose to return from a function. We can return a primitive type, an object literal or a function.
    What happens if there is no return statement in a function?
    Answer: if a function has no return type, it returns undefined by default.
    What do the apply() and call() methods do? What is the difference?
    Answer: The apply()and call() methods are used to set the execution context of the function by passing in the object reference as the first parameter.
